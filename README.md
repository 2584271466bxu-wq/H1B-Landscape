## Data Download

### 1. DOL LCA Disclosure Data
- URL: https://www.dol.gov/agencies/eta/foreign-labor/performance
- Download: H-1B LCA Disclosure Data (.xlsx), any years you want (FY2015–FY2026 recommended)
- Place in: `data/raw/lca/`

### 2. USCIS H-1B Employer Data Hub
- URL: https://www.uscis.gov/tools/reports-and-studies/h-1b-employer-data-hub
- Download: Employer Data Hub exports (.csv), all available years
- Place in: `data/raw/uscis/`

### 3. Reddit r/h1b (already downloaded)

## Run Data Pipelines

Run these **once** before launching the app. Order doesn't matter.

```bash
# From the h1b_project/ directory:
python data_processing/process_lca.py
python data_processing/process_uscis.py
python data_processing/process_reddit.py   # ~5–10 min at 20% comment sample
```

## Launch App

```bash
streamlit run app.py
```

The app works even if not all data files are ready — each page shows a placeholder with instructions if its data is missing.

---

## Deploy to Streamlit Community Cloud

1. Push this folder to GitHub (large raw data files go in `.gitignore`)
2. Go to https://share.streamlit.io → New app
3. Point to your repo, `app.py` as entry point
4. For the processed CSVs: either commit them directly (they're small) or host on Google Drive and fetch at runtime

## Key Design Decisions

- **Course color palette** used throughout (`#0072B2` default, `#FF3366` accent)
- **Plotly** for all interactive charts (consistent look, easy Streamlit integration)
- **VADER** over BERT: faster, no GPU needed, well-validated for social media short text
- **20% comment sample** default: processes ~200k comments in ~5 min vs 30+ min for full 1M
- Pages are **independent modules** — each imports its own data so you can develop them separately
